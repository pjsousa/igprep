# Role & Objective

You are a senior technical documentation specialist. Your task is to generate a **complete, production-ready `AGENTS.md` file** for an AI agent operating inside this repository.

The `AGENTS.md` file must allow any LLM agent to immediately understand the repository's purpose, structure, conventions, and how to contribute effectively — with zero prior context.

---

# Repository Context

This repository is a **personal preparation guide** for a **Tech Lead Engineer interview at IG Group**, a FTSE 100 fintech company. Key facts:

- **Target role:** Tech Lead Engineer (Java-focused)
- **Target company:** IG Group — a global fintech offering trading on stocks, futures, and CFDs to 1.3 million customers worldwide
- **Core technical focus:** High-frequency, high-performance Java code
- **Preparation style:** Structured study plan, live mock interviews, blog-style posts, and hands-on showcase projects

---

| Folder | Purpose |
|---|---|
| `mock_interviews/` | Transcripts and notes from live mock interview sessions conducted with an elite AI coach |
| `posts/` | Blog post series covering topics recommended by the elite coach for deeper preparation |
| `prompts/` | Collection of prompts used to build, explore, and iterate on content within this repo |
| `showcase_projects/` | Java projects demonstrating specific scenarios, solutions, and iterative improvements |

# Your Task

Generate a complete `AGENTS.md` file that includes all of the following sections:

1. **Repository Overview** — Purpose, target role, target company, and overall preparation strategy
2. **Repository Structure** — Annotated directory tree with a description of each folder's role
3. **Agent Operating Guidelines** — How an LLM agent should navigate, contribute to, and extend this repo
4. **Content Conventions** — Writing style, naming conventions, and formatting standards for each folder type (posts, mock interviews, prompts, showcase projects)
5. **Java & Technical Standards** — Coding conventions, performance expectations, and focus areas relevant to high-frequency Java development
6. **Contribution Workflow** — Step-by-step instructions for how an agent should add new content (e.g., a new post, a new showcase project, or a new mock interview entry)
7. **Goals & Success Criteria** — What "done well" looks like for this preparation repository

---

# Output Requirements

- Output **only** the `AGENTS.md` file content, formatted in **Markdown**
- Use clear headers, bullet points, and tables where appropriate
- Write in **third-person technical documentation style** (not conversational)
- Ensure the file is **self-contained** — an agent with no prior context should be fully operational after reading it
- Do **not** include any commentary, preamble, or explanation outside the Markdown file
